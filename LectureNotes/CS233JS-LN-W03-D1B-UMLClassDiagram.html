Unified Modeling Language

A way to visually represent software design models that is independent of any particular computer language. There are fourteen types of UML diagrams. We will just be using UML class diagrams.

  • Representing classes

    • Name

    • Fields (instance variables)

      • Access modifiers

        + public: can be accessed from outside the class

        # protected: can be accessed inside the class and in inherited classes

        − private: can only be accessed inside the class

    • Methods

  • Representing relationships

    • Association: arrow

    • Aggregation (equivalent to association): open diamond.

    • Composition: black diamond.

    • Inheritance: triangle.

  • Cardinality (multiplicity)

    • Ranges: indicate a range with two dots. For example: 0..n

    • Cardinality relationships

      • One-to-one: put 1 at the end with the diamond.

      • One-to-many: put 1..n at the end with the diamond—meaning the class with the diamond can have 1 to many instances of the other object.

      • Many-to-many: put diamonds on both ends of the line and put 1..n on both ends. It is good to avoid this relationship since it adds extra complexity.

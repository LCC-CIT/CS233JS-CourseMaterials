## Objectives

The objectives of this lab are:

1. Practice using asynchronous JavaScript features including `fetch`, `Promises`, and `async/await` syntax to communicate with external servers.
2. Practice querying, parsing, and rendering external JSON data payloads inside an existing application.
3. Learn how to interface with public RESTful Web APIs using dynamic URLs and request criteria.
4. Continue using modern development tools including Node.js, NPM, Vite, Git, and GitHub to manage and bundle your application.

### Study the ToDo List Example

Your instructor has provided an updated, completed **ToDo List** web app (included with the starter files) that demonstrates how to transition from completely local data storage into an API-driven application. Study this example carefully before updating your own app. Pay attention to:

- How network requests are initiated asynchronously inside a JavaScript module using the built-in `fetch()` API.
- How `async` and `await` operators are written to cleanly handle unresolved Promises.
- How incoming JSON responses are unpacked and conditionally integrated alongside existing `localStorage` data arrays.
- How the application UI reflects changes and displays content using `lit-html` templates once asynchronous data has resolved.

The updated ToDo List app makes an API request to a public, keyless motivational quote generator (such as the ZenQuotes API) to dynamically fetch a productivity quote at the top of the screen every time the dashboard loads.

### Complete Your App

Which app you enhance depends on your assignment group from Lab 4.  
Regardless of which app you are updating, for each one you will:

#### Version A: Web Page Bookmarker

Enhance your bookmarker application by integrating a call to the **Link Preview API** (`https://www.linkpreview.net/`). This integration allows the application to capture rich structural data about a web link instead of relying solely on raw text entries.

Your enhanced app must:
- Asynchronously call the `linkpreview.net` API whenever a user submits a new URL bookmark.
- Extract the rich **website title** and **preview image URL** returned by the API's JSON response payload.
- Update your data object so that both the text title and image URL are stored and preserved alongside the link in `localStorage`.
- Render your list of saved bookmarks as visual, rich graphical cards displaying the fetched preview image alongside the hyperlinked title.

#### Version B: Expense Tracker

Enhance your expense tracker application by integrating a call to the **SearchUPC Data API** (`https://searchupcdata.com/`). This enhancement brings automated commercial registry verification into your app, allowing items to be matched against global barcode data.

Your enhanced app must:
- Update your user input interface to accept a standardized product barcode number (UPC/EAN) or a text search parameter.
- Dispatch an asynchronous `fetch` request to the `searchupcdata.com` API endpoint using the user's provided search parameter.
- Automatically extract product naming details, category classifications, or estimated retail pricing information directly from the API response payload.
- Autocomplete your expense description entry using this fetched registry data, save the complete record inside `localStorage`, and display the enriched details clearly in the expense log list.

#### Version C: Grocery List

Enhance your grocery list application by integrating a call to the **Open Food Facts API** (`world.openfoodfacts.org/data`). This modification transforms your basic grocery checklist into an interactive, nutritionally-aware pantry manager.

Your enhanced app must:
- Send a dynamic `fetch` request to the `world.openfoodfacts.org` open database whenever a new food item or keyword is added to the list.
- Parse the resulting JSON response to extract secondary product descriptors, such as verified brand names, product thumbnails, or official health indicators (like the Nutri-Score classification).
- Render these rich visual indicators and product metadata cards directly next to each grocery item using updated `lit-html` templates.
- Ensure that the extended product data attributes are fully saved within your `localStorage` collection so that item profiles persist across sessions.

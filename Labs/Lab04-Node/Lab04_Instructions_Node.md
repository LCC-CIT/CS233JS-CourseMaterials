## Objectives

The objectives of this lab are:

1. Practice using Node.js, NPM, and Vite, which are dev tools commonly used in modern JavaScript development.
2. Practice using ES6 JavaScript features including classes, modules, and arrow functions.
3. Practice using Git and GitHub to manage your source code.

### Follow the ToDo List Example

Your instructor has provided a completed **ToDo List** web app (included in the starter files) that demonstrates how to structure a Vite-based JavaScript project. Study this example carefully before building your own app. Pay attention to:

- How the project files are organized.
- How JavaScript modules are imported and used.
- How the Vite dev server is started (`npm run dev`) and how the production build is generated (`npm run build`).

The ToDo List app lets a user add, complete, and remove tasks, and persists them using the browser's `localStorage`. Your app will follow a similar pattern.

### Development Workflow

For whichever version you build:

1. Install dependencies by running the following in the project's root folder:
   ```bash
   npm install
   ```
2. Start the Vite development server and test your app as you build it:
   ```bash
   npm run dev
   ```
3. When your app is complete, generate the production build:
   ```bash
   npm run build
   ```
4. Test your app in at least one browser using the Vite dev server on your local machine before submitting.

## Submission

### Beta Version and Code Review

- Share your beta version with a lab partner.
  - Your code should be at least 75% complete and working.
  - Share it in your lab partner channel on Discord by posting a <u>link to your GitHub repository</u>.
  - On Moodle, submit the link to your GitHub repository and report:
    - How complete was your web app — was it working?
    - Who did you share your beta code with?
    - What date and time did you share it?
- Commit and push your beta version to GitHub so your lab partner can view the code.
- Do a code review for your lab partner.
  - Post the completed review form on Discord.
  - Submit your code review to Moodle.
- Revise and improve your code based in part on feedback from your lab partner.

### Production Version

- Publish to the CIT web server.
  - Upload your completed web app to [citstudent.lanecc.edu](http://citstudent.lanecc.edu).
    Be sure to upload <u>only the files contained in the `dist` folder</u> (generated by `npm run build`).
  - Test the app on the web server to verify it still works there.

- Submit the following to Moodle:

  - A link to your GitHub repository in the online text.
  - The code review from your lab partner, with the "production" column filled out by you.
  - Screen shots showing[^1]:
    - Your project files in the VS Code file explorer and the terminal showing the Vite dev server running.
    - Your app running on the Vite dev server with the URL visible.
    - Your app running on citstudent with the URL visible.
  - The citstudent URL for your web app in the online text.



[^1]: Screenshots are requested for this lab so that you can show that you used Node.js and the Vite development tools introduced this week.
